Code notes and additional accessibility information

  • [1] html tag lang attribute specifies a document language.
  • [2] charset tag specifies a character set.
  • [3] viewport tag improves mobile device accessibility.
  • [4] meaningful and unique page title is required.
  • [5] skip to content anchor (visible only on focus).
  • [6] aria-label and aria-labelledby used to identify the sections of the web page.
  • [7] role="menubar" defines horizontal menu
  • [8] aria-haspopup="true" defines a submenu for the element exist
  • [9] aria-expanded="false" defines a state of the element (closed) and have to be changed with JS when needed
  • [10] role="menu" defines a menu
  • [11] role="menuitem" defines an item from a menu
  • [12] role="none" defines that native role will be discarded
  • [13] hidden attribute makes the inner links unfocusable
  • [14] single H1 tag on a page
  • [15] alt="" role="presentation" defines that the image is a decorative element
  • [16] title and aria-label attributes have to be the same otherwise the screen reader will read both of them
  • [17] role="alert" have to be dynamically added in order to announce the error immediately
  • [18] scope="col" specifies that the cell is a header for a column

Important things related to the visual

  • Fonts are large enough and readable.
  • There is enough contrast.
  • Color is not the only indicator to convey meaning.
  • Text wraps appropriately when resized to 200%.
  • Presented information is easy to understand.
  • Error handling (forms, etc.) is good enough, text is available
  • After high contrast is activated, everything is visible and interactable.
